# Distributed Lock

## 目标

训练分布式锁的正确实现、可重入性、看门狗机制、自动续期和故障恢复。

## 核心问题

- Redis 单实例锁和 Redlock 的区别？
- 为什么 SET NX EX 不够，还需要看门狗？
- 可重入锁在分布式环境下怎么实现？
- 锁的持有方崩溃后如何安全释放？
- ZooKeeper / etcd 实现锁与 Redis 实现有什么本质差异？
- 分布式锁和分布式事务的关系？

## 面试追问

- 如果 Redis 主从切换期间锁丢失，怎么解决？
- 如果业务执行时间超过锁过期时间，怎么办？
- 如果多个服务竞争同一把锁，如何避免惊群效应？
- 分布式锁的性能瓶颈在哪里？
