# Observability Platform

## 目标

训练可观测平台的设计：日志聚合、指标采集、分布式追踪、告警体系和数据可视化。

## 核心问题

- Log、Metric、Trace 如何统一采集和关联？
- 采样策略如何平衡成本和信息密度？
- 告警规则如何减少误报和漏报？
- 海量日志如何存储和检索？
- 追踪数据如何注入和传递（OpenTelemetry）？
- 如何设计告警分级和升级机制？

## 面试追问

- 如果系统产生 TB 级日志/天，如何降低存储成本？
- 如果链路追踪数据量过大，采样率怎么动态调整？
- 如何根据监控数据做根因分析（RCA）？
- 告警风暴时如何自动收敛？
