# Interview

## 核心追问

### 1. 库存是先扣还是预占？

**讨论**：预占库存防超卖，支付后真正扣减。

### 2. 状态机如何防错乱？

**讨论**：状态流转有日志，状态变更幂等。

### 3. 超时未支付如何处理？

**讨论**：定时任务扫描，释放预占库存。

### 4. 秒杀防超卖？

**讨论**：乐观锁 + 库存扣减条件 + 分桶。

### 5. 订单拆分（多商家）？

**讨论**：父订单 + 子订单，分布式事务。

### 6. 支付回调幂等？

**讨论**：支付渠道商口订单号唯一键。

## 生产踩坑

### 案例：库存扣减负数

现象：秒杀库存 100 件，200 人支付成功。

根因：并发 UPDATE 无库存校验。

教训：WHERE stock >= quantity 必须加。
