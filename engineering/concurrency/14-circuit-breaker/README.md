# 14 Circuit Breaker（熔断器）

## 问题描述

实现一个**熔断器（Circuit Breaker）**，保护下游服务免受过载请求压垮：
- **关闭（Closed）**：正常放行请求。如果错误率超过阈值，进入开启状态。
- **开启（Open）**：快速拒绝请求（返回错误），等待一段时间后进入半开状态。
- **半开（Half-Open）**：允许一个探测请求通过。如果成功，回到关闭状态；如果失败，回到开启状态。

## 核心概念

- **失败计数窗口**：基于连续失败数、错误率或滑动窗口统计。
- **恢复探测**：半开状态只允许有限探测，防止刚恢复的服务再次被压垮。
- **并发安全**：状态转换和计数必须在并发请求下正确。

## 约束

- 不得使用现成的熔断器库（如 Go 的 `gobreaker`、Python 的 `pybreaker`）。
- 必须显式实现三种状态及其转换逻辑。

## 手写提示

1. 用什么样的数据结构记录失败/成功？（连续计数 vs 滑动窗口 vs 时间桶）
2. 从开启到半开的转换由谁触发？（定时器？还是第一个到达的请求检查时间戳？）
3. 半开状态下，如何让"只有一个探测请求"通过？
4. 请求 panic / 超时了，算成功还是失败？

## 验证方式

```bash
make run
```

验证逻辑：模拟连续失败触发熔断、开启期快速失败、恢复探测、成功恢复关闭状态。
