# 13 Singleflight（请求合并）

## 问题描述

实现 **Singleflight**：当多个并发请求试图获取同一个资源（如缓存未命中后的数据库查询）时，只让第一个请求真正执行，其他请求等待并共享结果。

典型应用场景：防止**缓存击穿**（Cache Stampede）。

## 核心概念

- **请求去重**：相同 key 的并发请求合并为一次实际调用。
- **结果广播**：实际调用完成后，结果同时返回给所有等待的请求。
- **内存回收**：调用完成后，清理该 key 的等待队列。

## 约束

- 不得使用 Go 的 `golang.org/x/sync/singleflight` 等现成库。
- 必须处理并发 map 操作（锁或并发安全 map）。

## 手写提示

1. 用什么数据结构存储"进行中的调用"？（map[key]*call）
2. 当第二个请求到来时，如何让它等待第一个请求完成？（channel / condition variable / promise）
3. 第一个请求 panic 了怎么办？是否需要传播 panic 给所有等待者？

## 验证方式

```bash
make run
```

验证逻辑：并发发起 100 次相同 key 的请求，验证实际执行次数为 1，但 100 次请求都拿到了正确结果。
