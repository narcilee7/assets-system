# 03 Semaphore（信号量）

## 问题描述

实现一个**计数信号量（Counting Semaphore）**，允许最多 `N` 个执行单元同时进入临界区。

信号量是并发编程中最基础的原语之一，可用于：
- 限制并发连接数（连接池）
- 控制同时访问某资源的线程数
- 实现生产者-消费者等多种模式

## 核心概念

- **P 操作（Wait/Acquire）**：信号量值减 1；若为负，则阻塞等待。
- **V 操作（Signal/Release）**：信号量值加 1；若有等待者，唤醒一个。

## 约束

- 不得使用语言内置的 `Semaphore`（如 Python 的 `threading.Semaphore`、Go 的 `sync.WaitGroup` 等）。
- 必须自己维护计数器和等待队列。

## 手写提示

1. 信号量初始值代表可用资源数。
2. Acquire 时如果计数器已为 0，当前执行单元应阻塞。
3. Release 时应优先唤醒一个等待者，而非无条件增加计数器后让所有等待者竞争。

## 验证方式

```bash
make run
```

验证逻辑：启动远超 `N` 的任务，统计同时运行的最大并发数，必须不超过 `N`。
