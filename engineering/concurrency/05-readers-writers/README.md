# 05 Readers-Writers（读者-写者问题）

## 问题描述

这是操作系统课程中的经典同步问题：
- 多个读者可以同时读取共享数据。
- 写者必须独占访问（无读者、无其他写者）。
- **附加要求：写者优先**。当有写者等待时，新读者不能插队，防止写者饥饿。

## 核心概念

- **读者优先 vs 写者优先**：基础读写锁往往读者优先，导致写者饥饿。
- **公平性**：如何在读者并发性和写者响应性之间取得平衡？

## 约束

- 自行实现调度逻辑，不得直接使用现成的 `RWMutex`。
- 必须显式处理"写者优先"策略。

## 手写提示

1. 需要哪些额外状态来识别"有写者在等待"？
2. 新读者到达时，如果有写者等待，应该做什么？
3. 最后一个读者离开时，如果有写者等待，应该唤醒谁？
4. 写者完成后，应该优先唤醒写者还是读者？

## 验证方式

```bash
make run
```

验证逻辑：模拟高频读者 + 低频写者的场景，确保写者不会永远等待（通过超时检测）。
