# 04 Producer-Consumer（生产者-消费者）

## 问题描述

实现一个**有界缓冲区（Bounded Buffer）**：
- 多个生产者向缓冲区放入数据。
- 多个消费者从缓冲区取出数据。
- 缓冲区满时，生产者阻塞等待。
- 缓冲区空时，消费者阻塞等待。

## 核心概念

- **有界缓冲区**：防止生产者无限快于消费者导致内存溢出。
- **条件同步**：生产者和消费者需要互相通知（notFull / notEmpty）。

## 约束

- 不得使用语言内置的无界通道/队列直接作为缓冲区（如 Go 的无缓冲 channel 虽然天然适合，但本题要求自己管理缓冲区和条件变量）。
- 缓冲区容量由构造参数决定。

## 手写提示

1. 需要哪些状态？（buffer 切片/数组、头尾指针、当前元素数）
2. 生产者放入时需要等待什么条件？放入后需要通知谁？
3. 消费者取出时需要等待什么条件？取出后需要通知谁？
4. 是否需要互斥锁保护 buffer 本身？

## 验证方式

```bash
make run
```

验证逻辑：生产若干整数，消费者求和。最终总和必须正确，且过程中无 panic/exception。
