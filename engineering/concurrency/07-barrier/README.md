# 07 Barrier（线程屏障）

## 问题描述

实现一个**屏障（Barrier）**：N 个执行单元分阶段工作，每个阶段开始前必须等所有执行单元都到达屏障。只有全部到达后，大家才能一起进入下一阶段。

## 核心概念

- **分阶段同步**：MapReduce 的 shuffle 阶段、并行计算中的迭代算法都依赖屏障。
- **自旋 vs 阻塞**：高效实现应使用条件变量阻塞等待，而非 CPU 空转。
- **代（Generation）**：屏障需要区分不同轮次，防止上一轮的唤醒信号被本轮的等待者误收。

## 约束

- 不得使用语言内置的 `Barrier`（如 Python 的 `threading.Barrier`）。
- 必须自己实现计数、重置和唤醒逻辑。

## 手写提示

1. 如何区分第 1 轮屏障和第 2 轮屏障？（代/版本号机制）
2. 最后一个到达屏障的线程应该做什么？（重置计数、唤醒所有人）
3. 唤醒后，其他线程需要确认什么？（确认自己处于正确的代）

## 验证方式

```bash
make run
```

验证逻辑：每个执行单元记录每阶段的到达时间，验证同一阶段的到达时间全部一致（或非常接近）。
