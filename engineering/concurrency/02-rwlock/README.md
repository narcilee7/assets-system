# 02 RWLock（读写锁）

## 问题描述

实现一个读写锁（Read-Write Lock），满足：
- **多个读者可以同时持有读锁**。
- **写锁独占**：任何时刻最多只有一个写者，且写者持有锁时不能有读者。

## 核心概念

- **读共享、写独占**：读操作不修改状态，可并行；写操作修改状态，需独占。
- **写饥饿（Writer Starvation）**：如果读者源源不断，写者可能永远等待。本题先实现基础版本，公平性在 05-readers-writers 中深入。

## 约束

- 不得直接使用语言内置的 `RWMutex`（如 Go 的 `sync.RWMutex`）。
- 必须使用互斥锁 + 条件变量（或等价机制）自行实现。

## 手写提示

1. 需要哪些状态变量？（当前读者数、是否有写者、等待写者数……）
2. 读者进入和离开时需要做什么判断？
3. 写者获取锁时需要等待哪些条件？

## 验证方式

```bash
make run
```

验证逻辑：启动多个读者和写者，确保写者执行期间没有活跃读者，且多个读者可以同时活跃。
