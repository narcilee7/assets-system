# 12 Context Cancellation（上下文取消）

## 问题描述

实现一个可取消的**并行任务组**：
- 启动多个长时间运行的子任务（如定时打印）。
- 当任意一个子任务出错，或收到外部取消信号时，**所有其他子任务应立即停止**。
- 父任务需要等待所有子任务真正退出后才能返回。

## 核心概念

- **取消传播**：取消信号应像树一样从根节点传播到所有叶子节点。
- **协作式取消**：子任务需要定期检查取消状态（轮询 channel / 检查事件 / 捕获异常）。
- **不泄漏 goroutine / 线程**：父退出前必须确认子已退出。

## 约束

- 不得使用 Go 的 `context.Context`、Python 的 `asyncio.CancelledError` 等内置取消机制（了解其原理后自己实现简化版）。
- 必须显式管理取消 channel / 事件和 WaitGroup。

## 手写提示

1. 取消信号用什么数据结构传递？（channel、Event、AbortController）
2. 子任务如何"优雅地"响应取消？（在阻塞操作前检查、设置超时）
3. 如果子任务正在执行一个无法中断的阻塞调用怎么办？

## 验证方式

```bash
make run
```

验证逻辑：启动任务，触发取消，验证所有子任务在超时前退出，且无泄漏。
