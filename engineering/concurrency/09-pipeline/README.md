# 09 Pipeline（流水线）

## 问题描述

实现一个**三阶段流水线**：
1. **Generate**：生成整数序列（1 到 N）。
2. **Square**：接收整数，计算平方。
3. **Sum**：接收平方值，求和。

每个阶段通过 channel / queue 连接，前一阶段的输出作为后一阶段的输入。阶段之间解耦，可以独立扩展或替换。

## 核心概念

- **阶段解耦**：每个阶段只关心输入和输出格式，不依赖其他阶段的实现细节。
- **背压（Backpressure）**：如果下游处理慢，上游应阻塞（channel/queue 的天然特性）。
- **优雅退出**：当上游没有更多数据时，下游应能正确结束。

## 约束

- 不得使用语言内置的流处理库（如 Python 的 `multiprocessing.Pool` 的 map）。
- 必须显式管理阶段间的连接和关闭。

## 手写提示

1. 每个阶段在什么情况下关闭自己的输出 channel / 队列？
2. 如果中间阶段 panic 了，如何传播错误/关闭整个流水线？
3. 能否让 Square 阶段并行处理多个数？（提示： fan-out，见下一题）

## 验证方式

```bash
make run
```

验证逻辑：计算 1 到 100 的平方和，结果应为 338350。
