# 01 Mutex and Lock（互斥锁）

## 问题描述

实现一个线程安全的计数器 `SafeCounter`。启动 N 个并发执行单元，每个对计数器递增 M 次。最终计数器的值必须等于 N × M。

## 核心概念

- **临界区（Critical Section）**：访问共享资源的代码段。
- **互斥（Mutual Exclusion）**：任何时刻最多只有一个执行单元进入临界区。
- **竞态条件（Race Condition）**：缺少互斥保护时，执行结果依赖于执行单元的调度顺序。

## 约束

- 不得使用语言内置的原子类型（如 Go 的 `atomic.AddInt64`）或现成的线程安全计数器。
- 必须显式使用锁（Lock / Mutex）来保护共享状态。

## 手写提示

1. 锁的粒度：是保护整个 `Inc()` 方法，还是只保护 `count++` 这一行？
2. 锁的持有时间：尽量短，但也要保证逻辑正确。
3. 如果 `Value()` 也需要锁保护，是同一个锁还是不同的锁？
