# 11 Rate Limiter — Token Bucket（令牌桶限流器）

## 问题描述

实现一个**令牌桶（Token Bucket）**限流器：
- 桶有固定容量 `capacity`。
- 以固定速率 `rate`（每秒令牌数）向桶中添加令牌。
- 每个请求尝试从桶中取出 1 个令牌；如果桶空，请求被限流（返回 false）。

## 核心概念

- **令牌桶 vs 漏桶**：令牌桶允许突发流量（桶中有积累），漏桶强制匀速。
- **懒加载（Lazy Refill）**：不需要定时器，而是在每次请求时根据时间差计算应添加的令牌数。

## 约束

- 必须支持并发访问（多个执行单元同时调用 `Allow()`）。
- 不得使用定时器 goroutine / 线程主动填充令牌（使用懒加载）。

## 手写提示

1. 需要记录哪些状态？（当前令牌数、上次填充时间、容量、速率）
2. 如何原子地更新令牌数和上次填充时间？
3. 浮点数计算是否有精度问题？是否需要用整数表示（如每毫秒产生的令牌数）？

## 验证方式

```bash
make run
```

验证逻辑：以固定间隔发送请求，验证在不超过突发容量时允许，持续超额时拒绝。
