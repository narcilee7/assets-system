# RBAC Permission Model

## 目标

训练基于角色的访问控制（RBAC）系统的设计与实现。包括用户、角色、权限、资源和操作的完整权限模型。

## 核心概念

| 概念 | 解释 |
| --- | --- |
| User | 系统用户 |
| Role | 角色（权限的集合） |
| Permission | 权限（资源 + 操作） |
| UserRoleMapping | 用户角色关联 |
| RolePermissionMapping | 角色权限关联 |

## 核心功能

- **角色分配**：用户可以被分配多个角色
- **权限继承**：角色拥有权限，用户通过角色获得权限
- **权限检查**：检查用户是否有执行某个操作的权限
- **角色层级**：支持角色继承（如 admin > editor > viewer）

## 边界条件

- **用户多角色**：用户可以有多个角色，权限取并集
- **角色层级**：子角色继承父角色的所有权限
- **权限冲突**：同资源多角色可能产生冲突
- **最小权限原则**：默认无权限，需显式授权

## 面试追问

- RBAC 和 ABAC 的区别？
  （答：RBAC 基于角色，适合相对静态的权限管理；ABAC 基于属性，更细粒度但更复杂。）
- 如何实现权限的委托？
  （答：通过角色继承，或者专门的委托机制。）
- 权限变更如何生效？
  （答：通常通过缓存失效，或者实时检查权威数据源。）
