# 11 Chat Server（聊天服务器）

## 问题描述

实现一个 **TCP 聊天服务器**：
- 客户端连接后，需要发送用户名进行注册。
- 支持**广播消息**：一个客户端发送的消息会被转发给所有其他在线客户端。
- 支持**在线列表**：客户端发送 `/list` 命令，返回当前所有在线用户。
- 客户端断开时，从在线列表中移除，并通知其他人。

## 核心概念

- **连接管理**：维护所有活跃连接的集合，需要并发安全地增删查。
- **广播**：遍历连接集合，逐条发送消息。注意：发送时如果某个客户端阻塞，不应影响其他客户端。
- **用户状态**：用户名与连接的映射，断开时的清理逻辑。

## 约束

- 不得使用现成的聊天/消息队列框架。
- 必须基于原始 TCP socket 实现协议解析和消息路由。

## 手写提示

1. 广播时是否需要加锁保护连接集合？如何避免在遍历过程中修改集合？
2. 如果某个客户端接收缓慢，广播是否会阻塞服务器？（考虑使用发送队列+独立 goroutine）
3. 如何处理客户端异常断开（没有发退出命令直接断网）？

## 验证方式

```bash
make run
```

验证逻辑：多个 client 连接，注册用户名，发送消息，验证广播和在线列表功能正确。
