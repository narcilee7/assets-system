# 09 Proxy（TCP 反向代理）

## 问题描述

实现一个 **TCP 反向代理**：
- 监听一个端口，接收客户端连接。
- 为每个客户端连接建立一个到后端服务器的连接。
- 在客户端和后端之间**双向转发**数据。
- 当任意一方关闭连接时，关闭另一方。

## 核心概念

- **透明转发**：代理不修改数据内容，只是搬运字节。
- **双向复制**：需要同时从 client->backend 和 backend->client 两个方向复制数据。
- **连接生命周期**：任意一方断开，代理应立即断开另一方，避免资源泄漏。

## 约束

- 不得使用语言内置的代理库或高级转发封装。
- 必须显式管理两端 socket 的读写和关闭。

## 手写提示

1. 如何实现双向并发复制？（两个 goroutine / 两个线程 / pipe）
2. 当一方断开时，如何通知另一方？（关闭对端 socket 会导致对端 read 返回 0 或出错）
3. 如果后端服务器暂时不可用，代理应如何处理？（立即拒绝连接 or 重试）

## 验证方式

```bash
make run
```

验证逻辑：启动后端 echo server，启动代理，client 通过代理连接并发送消息，验证收到 echo 回复。
