# 03 IO Multiplexing（IO 多路复用）

## 问题描述

实现一个 **单线程 TCP 服务器**，能够同时处理多个客户端连接，不使用"一个连接一个线程/进程"模型。

- 使用 **IO 多路复用**（`select` / `poll` / `epoll`）监控多个 socket 的可读状态。
- 当某个连接有数据到达时，读取并 echo 回去。
- 当某个连接关闭时，从监控集合中移除。

## 核心概念

- **select/poll/epoll**：让单个执行单元同时等待多个 fd 的就绪事件。
- **单线程并发**：没有线程切换开销，但需要非阻塞或配合多路复用。
- **C10K 问题**：当连接数达到万级时，"一个连接一个线程"模型不可扩展。

## 约束

- **Python**：必须使用 `select.select` 显式调用。
- **Node.js**：基于 `net` 模块的事件驱动，但要求**手动维护连接集合**，不使用框架。
- **Go**：使用单个 goroutine + `select` 语句配合 channel 模拟多路复用，或直接使用 `epoll` 系统调用。

## 手写提示

1. `select` 的 fd 集合大小有限制（通常是 1024），`epoll` 没有这个限制。
2. 如何处理新连接（listener 可读）和现有连接（client 可读）？
3. 客户端断开时，`read` 返回 0 还是抛出异常？如何正确关闭和移除 fd？

## 验证方式

```bash
make run
```

验证逻辑：启动 server，多个 client 同时连接并发送消息，验证每个 client 都能收到 echo 回复。
