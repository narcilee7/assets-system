# 08 Connection Pool（连接池）

## 问题描述

实现一个 **TCP 连接池**：
- 维护一组可复用的 TCP 连接到后端服务器。
- 支持最大连接数限制，超过时阻塞等待或返回错误。
- 支持连接超时（获取连接超时、空闲连接超时）。
- 支持连接的创建、复用、健康检查和关闭。

## 核心概念

- **连接复用**：避免频繁创建/销毁 TCP 连接的三次握手和四次挥手开销。
- **池化资源**：最小连接数、最大连接数、空闲超时驱逐。
- **获取等待**：当池满时，请求可以阻塞等待、立即失败或创建临时连接。

## 约束

- 不得使用语言内置的连接池（如 Go 的 `database/sql` 连接池、Python 的 `urllib3` 连接池）。
- 必须自己管理连接集合、锁和超时逻辑。

## 手写提示

1. 连接池需要什么数据结构？（可用连接队列、正在使用集合、最大容量）
2. 如何确保连接是健康的？（获取时检查 `conn.Write` 心跳或 `SetReadDeadline`）
3. 空闲连接多久应该关闭？（定时器扫描 or 懒检查）
4. 连接池关闭时，如何确保所有连接都被正确关闭？

## 验证方式

```bash
make run
```

验证逻辑：启动后端 server，从池中并发获取连接发送消息，验证连接被复用且总数不超过上限。
