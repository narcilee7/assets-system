# 14 Reliable UDP（可靠 UDP 传输）

## 问题描述

在 UDP 之上实现一个**简化版的可靠传输协议**：
- **序列号（Sequence Number）**：每个数据包带序列号，接收方按序交付。
- **ACK（确认）**：接收方收到包后发送 ACK。
- **超时重传**：发送方如果在超时内未收到 ACK，重传该包。
- **滑动窗口**：发送方可以连续发送多个包而不等待每个 ACK（简化版固定窗口）。

## 核心概念

- **不可靠网络**：UDP 不保证到达、顺序、不重复。
- **停等协议（Stop-and-Wait）**：发一个包，等 ACK，再发下一个（简化版）。
- **累积 ACK**：ACK N 表示 N 及之前的所有包都已收到。
- ** Karn 算法**：重传后收到 ACK，不能简单更新 RTT（本题简化处理）。

## 约束

- 必须使用 UDP socket，不得使用 TCP。
- 不得使用现成的可靠 UDP 库（如 QUIC、ENet、KCP）。
- 单包数据不超过 1024 字节。

## 手写提示

1. 包格式怎么设计？（Header: seq + ack + flags，Payload: data）
2. 超时时间设为多少合适？（固定值 or 动态 RTT 计算）
3. 如果 ACK 丢了，发送方会重传，接收方收到重复包怎么办？
4. 如何优雅地结束传输？（发送 EOF 标志包并确认）

## 验证方式

```bash
make run
```

验证逻辑：发送方发送一段文本，接收方完整接收并按序重组，验证内容一致。
