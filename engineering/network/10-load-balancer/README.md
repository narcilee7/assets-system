# 10 Load Balancer（负载均衡器）

## 问题描述

实现一个 **TCP 负载均衡器**：
- 监听一个端口，接收客户端连接。
- 使用 **Round Robin** 算法将连接分发到多个后端服务器。
- 支持简单的**健康检查**：如果某个后端连续失败 N 次，将其标记为不可用；每隔一段时间尝试恢复探测。

## 核心概念

- **负载均衡算法**：Round Robin（轮询）、Least Connections（最少连接）、IP Hash（一致性哈希）。
- **健康检查**：主动探测（定时连接后端）或被动探测（根据请求失败率）。
- **故障转移**：后端不可用时，快速切换到可用后端。

## 约束

- 不得使用现成的负载均衡器或反向代理库。
- 必须自己实现 Round Robin 调度逻辑和健康检查。

## 手写提示

1. Round Robin 的计数器是否需要加锁保护？
2. 健康检查是同步阻塞的还是异步的？如何避免健康检查影响正常请求？
3. 所有后端都不可用时，负载均衡器应该如何行为？（拒绝连接 or 排队等待）

## 验证方式

```bash
make run
```

验证逻辑：启动多个后端 server，启动负载均衡器，发送请求，验证流量按轮询分发；关闭一个后端，验证流量只分发给剩余健康后端。
