# 02 Virtual List（虚拟列表）

## 问题描述

实现一个虚拟列表组件 `VirtualList`，在大量数据（10000+ 条）下只渲染可见区域的 DOM 节点，通过滚动动态回收和复用 DOM。

## 核心概念

- **视口（Viewport）**：用户可见的区域。
- **起始索引（startIndex）**：当前视口第一个可见项的索引。
- **缓冲（buffer）**：在可见区域上下额外渲染的项数，提升滚动流畅度。
- **项高（itemHeight）**：固定行高（可用估计高度 + 真实高度测量优化）。

## 约束

- 数据源为纯数组，不限长度。
- 只在视口内渲染 DOM，超出部分为空白占位。
- 支持动态高度（通过测量）。
- 滚动时及时回收不可见节点并创建新节点。

## 手写提示

1. 如何计算 `startIndex`？用 `scrollTop / itemHeight` 估算。
2. 如何处理动态高度？维护一个 `positions[]` 数组记录每项的 offset。
3. 容器高度 = 所有项高度之和（缓存）。
4. 事件防抖：监听 scroll 事件，计算 visible range，触发 render。
