# 微前端架构基础

## 1. 什么是微前端

```
微前端 = 将前端应用拆分为多个独立部署的子应用
         由主应用（壳）集成，对用户呈现为单一应用

┌─────────────────────────────────────────┐
│              主应用 (Shell)               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 子应用 A │  │ 子应用 B │  │ 子应用 C │ │
│  │ React   │  │ Vue     │  │ Angular │ │
│  │ Team 1  │  │ Team 2  │  │ Team 3  │ │
│  └─────────┘  └─────────┘  └─────────┘ │
│         独立仓库 / 独立部署 / 独立 CI/CD   │
└─────────────────────────────────────────┘
```

## 2. 决策条件

### 适合微前端

| 场景 | 说明 |
|------|------|
| 多团队独立交付 | 各团队有自己的发布节奏，不想互相阻塞 |
| 技术栈迁移期 | 存量系统逐步迁移到新框架 |
| 子应用生命周期独立 | 某些模块可以独立上线/下线 |
| 存量系统整合 | 多个独立系统需要整合为统一门户 |
| 巨型应用拆分 | 构建时间 > 5min，Bundle > 1MB |

### 不适合微前端

| 场景 | 说明 |
|------|------|
| 单团队小应用 | 复杂度不划算 |
| 强一致交互 | 频繁跨应用通信，耦合严重 |
| 性能预算紧 | 微前端有运行时开销 |
| 只是为了拆目录 | 用 Monorepo 即可 |

## 3. 架构模式对比

| 模式 | 隔离性 | 集成度 | 复杂度 | 适用 |
|------|--------|--------|--------|------|
| **iframe** | 最强 | 最低 | 低 | 完全独立的第三方应用 |
| **qiankun/single-spa** | 强 | 高 | 中 | 同域多技术栈 |
| **Module Federation** | 弱 | 最高 | 中 | 同构建工具、共享依赖 |
| **Web Components** | 强 | 高 | 中 | 框架无关组件 |
| **构建时集成** | 无 | 最高 | 低 | 过渡阶段、简单场景 |

## 4. 核心挑战

```
┌─────────────────────────────────────────────────────────────┐
│                      微前端核心挑战                          │
├──────────────┬──────────────────────────────────────────────┤
│ 路由         │ 主应用路由 vs 子应用路由，浏览器历史管理       │
├──────────────┼──────────────────────────────────────────────┤
│ 样式隔离     │ 全局 CSS 污染、CSS 变量冲突                     │
├──────────────┼──────────────────────────────────────────────┤
│ JS 隔离      │ 全局变量污染、window 挂载、事件监听泄漏          │
├──────────────┼──────────────────────────────────────────────┤
│ 状态共享     │ 登录态、用户信息、主题、语言                     │
├──────────────┼──────────────────────────────────────────────┤
│ 公共依赖     │ React/Vue 重复加载、版本冲突、Bundle 膨胀        │
├──────────────┼──────────────────────────────────────────────┤
│ 通信         │ 跨应用事件、数据传递、方法调用                    │
├──────────────┼──────────────────────────────────────────────┤
│ 部署         │ 独立发布、版本协调、回滚策略                      │
└──────────────┴──────────────────────────────────────────────┘
```
