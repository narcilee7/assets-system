# 部署策略

## 1. 蓝绿部署

```
┌─────────────┐      ┌─────────────┐
│   Blue      │      │    Green    │
│  (当前版本)  │      │  (新版本)   │
└──────┬──────┘      └──────┬──────┘
       │                     │
       └────────┬────────────┘
                │
           Load Balancer
                │
         流量 100% → Blue
         验证 Green
         切换 → 100% Green
```

**优点**：零停机、快速回滚  
**缺点**：双倍资源

## 2. 金丝雀发布

```
时间线：
  t0: 100% v1
  t1:  95% v1 + 5% v2   ← 灰度开始
  t2:  90% v1 + 10% v2
  ...
  tn:   0% v1 + 100% v2  ← 完全切换

监控指标：
  - 错误率 < 阈值
  - P95 延迟 < 阈值
  - 业务转化率无异常
```

## 3. 滚动部署

```
逐步替换实例：
  Pod 1: v1 → v2（验证健康）
  Pod 2: v1 → v2
  Pod 3: v1 → v2
  ...
```

## 5. 策略对比

| 策略 | 风险 | 回滚速度 | 资源 | 适用 |
|------|------|----------|------|------|
| 蓝绿 | 低 | 瞬间 | 2x | 关键核心服务 |
| 金丝雀 | 中 | 快 | 1x | 大部分场景 |
| 滚动 | 中 | 中 | 1x | K8s 默认 |
| Feature Flag | 低 | 瞬间 | 1x | 功能灰度 |
