# 测试策略

## 1. 测试金字塔

```
        ┌─────────┐
        │   E2E   │  ← 慢、贵、高置信度
        │  10%    │     用户旅程、关键路径
        ├─────────┤
        │Component│  ← 中速、中成本
        │  30%    │     组件交互、UI 状态
        ├─────────┤
        │  Unit   │  ← 快、便宜、低置信度
        │  60%    │     纯函数、工具、hooks
        └─────────┘
```

## 2. 测试决策树

```
测试什么？
  ├─ 纯函数 / 工具 → 单元测试
  ├─ React/Vue 组件 → 组件测试（Testing Library）
  ├─ 页面路由 / API 集成 → E2E（Playwright）
  ├─ UI 样式变化 → 视觉回归
  └─ 业务逻辑复杂 → 单元 + 集成

不要测试什么？
  ├─ 第三方库（已测试）
  ├─ 简单的样式（视觉回归覆盖）
  └─ 静态类型（TypeScript 覆盖）
```

## 3. 测试指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 行覆盖率 | > 80% | 不是越高越好 |
| 分支覆盖率 | > 70% | 比行覆盖更重要 |
| 测试速度 | < 5min | CI 反馈速度 |
|  flaky 率 | < 1% | 不稳定的测试有害 |
