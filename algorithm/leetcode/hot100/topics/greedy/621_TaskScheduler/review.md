# LeetCode 621 — Task Scheduler

## 模式
贪心 / 优先队列 / 数学

## 识别信号
- 安排任务，最小时间间隔 n
- 冷却期
- 最多任务优先

## 边界
- 只有一种任务
- 冷却期大于任务种类
- 所有任务冷却中

## 实现要点
- 优先队列按频次取任务
- 分轮次：每轮放最多任务，间隔用 idle 填充
- 数学：max = maxFreq - 1) * (n + 1) + countMaxFreq
- 取 max(数学公式结果, 总任务数)

## 复杂度
- 时间: O(N)
- 空间: O(1) 或 O(26)

## 追问
- 公式怎么来的？→ 最长频次任务占位，间隔插入其他任务
- 为什么取 max？→ 冷却期可能用不上
- 优先队列适合什么场景？→ 需要动态选择最高频次任务
