# LeetCode 406 — Queue Reconstruction by Height

## 模式
贪心

## 识别信号
- 按 (h, k) 排序后重建队列
- h 为身高，k 为前面身高 >= h 的人数
- 按身高降序、k 升序排序后插入

## 边界
- 所有人身高相同
- k 为 0
- k 等于人数

## 实现要点
- 按 h 降序，k 升序排序
- 遍历，按 k 位置插入结果队列
- 高个子先排，低个子插入时前面位置已确定

## 复杂度
- 时间: O(n^2) 插入最坏
- 空间: O(n)

## 追问
- 为什么排序后插入即正确？→ 高个子不影响低个子的 k
- 如果身高升序？→ 不对，需要先排高个子
- 如果要返回坐标？→ 插入后返回索引
