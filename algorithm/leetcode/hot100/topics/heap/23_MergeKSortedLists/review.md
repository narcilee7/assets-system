# LeetCode 23 — Merge k Sorted Lists

## 模式
堆（优先队列） / 分治

## 识别信号
- K 个有序链表合并
- 多路归并
- 类似归并排序的 merge 阶段

## 边界
- k = 0 → null
- 所有链表为空
- 只有一个链表

## 实现要点
- 堆: 维护大小为 k 的最小堆，每次弹出最小节点
- 分治: 两两合并，层层归约
- 堆时间 O(n log k)

## 复杂度
- 时间: O(n log k) 堆 或 O(n) 分治
- 空间: O(1) 分治，O(k) 堆

## 追问
- 分治和堆哪个更好？→ 分治更均衡
- 如果 k=2？→ 21 题
- 如果要找中位数？→ 合并后二分
