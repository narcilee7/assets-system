# LeetCode 234 — Palindrome Linked List

## 模式
快慢指针 / 链表

## 识别信号
- 判断链表是否回文
- O(n) 时间 + O(1) 空间
- 找中点 + 反转 + 比较

## 边界
- 单节点 → true
- 两节点 → 比较值
- 奇偶长度

## 实现要点
- slow/fast 找中点
- 反转前半部分
- 比较前半和后半（偶数）或前半和后半去掉中点（奇数）

## 复杂度
- 时间: O(n)
- 空间: O(1)

## 追问
- 为什么不用额外空间存数组？→ O(n) 空间
- 如果节点值比较无法区分？→ 用结构而非值
- 奇偶长度处理有何不同？→ 奇数时 slow 多走一步
