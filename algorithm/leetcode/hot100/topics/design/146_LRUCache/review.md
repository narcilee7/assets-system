# LeetCode 146 — LRU Cache

## 模式
哈希 + 链表 / 设计

## 识别信号
- 最近最少使用缓存
- 需要 O(1) get 和 put
- 淘汰最久未使用的

## 边界
- 容量已满时淘汰
- 访问或插入都更新顺序
- 连续访问同一 key

## 实现要点
- 双向链表维护访问顺序（头部最新，尾部最老）
- 哈希表存储 key → 节点
- get 时移动到头部
- put 时新节点插入头部，满则删尾部

## 复杂度
- 时间: O(1)
- 空间: O(capacity)

## 追问
- LFU 和 LRU 的区别？→ LFU 考虑访问频率
- 如果要线程安全？→ 加锁或并发数据结构
- LinkedHashMap 直接实现？→ Java 可直接用
