# LeetCode 72 — Edit Distance

## 模式
动态规划

## 识别信号
- 两字符串最小编辑距离
- 可插入、删除、替换
- 操作数最小

## 边界
- 两字符串相等 → 0
- 任一为空 → 另一字符串长度
- 完全不同

## 实现要点
- dp[i][j] = word1 前 i 个变成 word2 前 j 个的最小操作
- dp[i][j] = min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1]+(w1[i-1]==w2[j-1]?0:1))
- 删除=dp[i-1][j]+1，插入=dp[i][j-1]+1，替换=dp[i-1][j-1]+1

## 复杂度
- 时间: O(mn)
- 空间: O(mn) 或 O(n) 优化

## 追问
- 如果限制只能用 k 次操作？→ 检查 dp[m][n] <= k
- 如果要求返回具体操作？→ 记录转移路径
- 如果是求最长公共子序列？→ dp[i][j] - 替换成本
