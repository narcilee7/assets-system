# 训练诊断清单

模型不收敛、loss 震荡、验证集不升——这些问题需要系统化的诊断方法。

## 训练曲线诊断

### 正常训练

```
Train Loss: 平滑下降 → 收敛
Val Loss:   先降后升（轻微过拟合）或同步收敛
```

### 异常模式与对策

| 曲线模式 | 可能原因 | 解决 |
|---------|---------|------|
| Loss 不下降 | lr 太小、梯度消失、数据问题 | 增大 lr、检查激活函数、检查数据 |
| Loss 爆炸 (NaN) | lr 太大、梯度爆炸、数值问题 | 梯度裁剪、减小 lr、检查输入归一化 |
| Loss 震荡剧烈 | lr 太大、batch size 太小 | 减小 lr、增大 batch、加 momentum |
| Train ↓ Val ↑ | 过拟合 | 正则化、dropout、早停、更多数据 |
| Train Val 都高 | 欠拟合 | 增大模型、减小正则、更久训练 |
| Loss 阶梯状下降 | lr 太大，在损失盆地边缘跳跃 | 减小 lr、用余弦衰减 |

## 检查清单

### 数据层面

- [ ] 输入数据是否归一化/标准化？
- [ ] 标签是否正确？（随机抽查几个样本）
- [ ] 是否存在数据泄露（train/val 重叠）？
- [ ] 类别是否平衡？不平衡时是否用 weighted loss？
- [ ] 数据增强是否合理？（不要增强到面目全非）

### 模型层面

- [ ] 输出层激活函数是否匹配任务？（分类用 softmax，回归用线性）
- [ ] 损失函数是否正确？（多分类用 CrossEntropy，不要先 softmax 再 BCE）
- [ ] 初始化是否合理？（深层用 Xavier/He，不要用默认小随机数）
- [ ] BatchNorm 位置是否正确？（通常在激活函数前）

### 优化层面

- [ ] 学习率是否合适？（从 1e-3 开始扫描）
- [ ] 是否用了 warmup？（Transformer 必须）
- [ ] 梯度裁剪是否设置？（RNN/Transformer 推荐）
- [ ] 混合精度是否导致 NaN？（尝试 full precision 对比）

### 工程层面

- [ ] 随机种子固定了吗？（复现性）
- [ ] 数据加载器是否有 shuffle？（训练必须 shuffle）
- [ ] 梯度是否累积正确？（多步累积时除以步数）
- [ ] 分布式训练是否同步正确？

## 快速调试流程

```
1. 小数据过拟合测试
   - 用 10-100 个样本训练
   - 目标：train loss → 0
   - 如果做不到 → 模型/代码 bug

2. 检查损失值范围
   - 随机初始化模型的 loss 应接近理论上限
   - 例如：10 类分类，随机猜的 CE ≈ ln(10) ≈ 2.3

3. 可视化中间结果
   - 第一层权重是否学到边缘/纹理？
   - Attention map 是否合理？

4. 逐步增加复杂度
   - 先跑通小模型 + 小数据
   - 再逐步增加数据和模型大小
```

## 常见误区

| 误区 | 正解 |
|------|------|
| Loss 不再降就停止 | ❌ 可能卡在平台期，尝试 lr decay |
| 验证 loss 最低 = 最好模型 | ⚠️ 通常是的，但有时稍高的 val loss 泛化更好（flat minima） |
| 大 batch 一定更好 | ❌ 大 batch 需要大 lr，泛化可能变差 |
| 数据增强越多越好 | ❌ 过度增强会让模型学到错误模式 |

## 快速检查清单

- [ ] 做过小数据过拟合测试
- [ ] 检查过随机初始化的 loss 是否合理
- [ ] 可视化过至少一层中间特征
- [ ] 监控过梯度范数（防止爆炸/消失）
