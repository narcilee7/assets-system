# 树模型与集成学习

树模型是结构化数据的王者。从单棵决策树到 XGBoost、LightGBM，它们在表格数据上依然是深度学习的有力竞争者。

## 决策树

### 分裂准则

| 任务 | 准则 | 公式 |
|------|------|------|
| 分类 | 基尼不纯度 (Gini) | `G = 1 - Σ p_i²` |
| 分类 | 信息增益 (Entropy) | `IG = H(parent) - Σ (n_i/n) H(child_i)` |
| 回归 | 方差减少 (MSE) | `Σ (y_i - ȳ)²` |

### 树的剪枝

| 策略 | 方式 | 效果 |
|------|------|------|
| 预剪枝 | 限制深度、最小叶子样本数 | 简单，可能欠拟合 |
| 后剪枝 | 先长完整树，再合并叶子 | 效果更好，计算量大 |

## 集成方法

### Bagging (Bootstrap Aggregating)

```
1. 从训练集有放回采样 N 次，生成 M 个子集
2. 每个子集训练一个基学习器
3. 预测时投票（分类）或平均（回归）
```

**随机森林 (Random Forest)** = Bagging + 特征随机子集
- 降低方差，不易过拟合
- 并行训练，速度快

### Boosting

```
1. 顺序训练弱学习器
2. 每轮关注前一轮分错的样本
3. 加权组合所有学习器
```

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| AdaBoost | 调整样本权重 | 简单任务 |
| Gradient Boosting | 拟合残差 | 中小数据 |
| XGBoost | 二阶泰勒展开、正则化、并行 | 竞赛标配 |
| LightGBM | 直方图算法、Leaf-wise 生长 | 大数据，速度快 |
| CatBoost | 有序提升、自动处理类别特征 | 类别特征多的数据 |

### XGBoost 核心

```
Obj = Σ L(y_i, ŷ_i) + Σ Ω(f_k)
      损失函数      +  正则项（叶子权重 L2 + 叶子数）

用二阶泰勒展开近似：
g_i = ∂L/∂ŷ_i      (一阶梯度/残差)
h_i = ∂²L/∂ŷ_i²    (二阶梯度/Hessian)
```

- 贪心选择最佳分裂点
- 支持列采样、行采样防止过拟合
- 内置处理缺失值

## 与深度学习的对比

| 维度 | 树模型 (XGBoost) | 神经网络 |
|------|------------------|---------|
| 数据类型 | 表格数据王者 | 图像、文本、序列 |
| 特征工程 | 需要但要求低 | 需要更多（embedding） |
| 训练速度 | 快 | 慢（尤其大模型） |
| 解释性 | 强（特征重要性） | 弱（黑盒） |
| 外推能力 | 弱（只能插值） | 相对强 |
| 大数据 | LightGBM 可扩展 | 天然适合 |

## 常见误区

| 误区 | 正解 |
|------|------|
| 树模型不需要调参 | ❌ 学习率、深度、子采样都影响很大 |
| Boosting 不会过拟合 | ❌ 迭代次数过多会过拟合，需早停 |
| 特征重要性高 = 因果关系 | ❌ 只是相关性，可能有混杂因素 |
| 随机森林比 XGBoost 强 | ❌ 通常 Boosting 在竞赛中更好 |
