# Chain-of-Thought (CoT) 推理

Chain-of-Thought 是让 LLM 展示推理过程的核心技术，是提升复杂任务性能的关键手段。

## 核心思想

```
标准 prompting：
Q: 一个农场有鸡和兔，头共 35 个，脚共 94 只。鸡兔各多少？
A: [直接输出答案，容易错]

CoT prompting：
Q: ...
A: 让我一步一步思考。首先...然后...所以...
[展示推理过程，答案更准确]
```

##  prompting 方法

### Zero-shot CoT

```
在问题后加："Let's think step by step."
或中文："让我们一步一步思考。"

无需示例，模型自动产生推理过程
```

### Few-shot CoT

```
提供包含推理过程的示例：

Q: 1+2+3+...+10 = ?
A: 1+10=11, 2+9=11...共 5 对，所以 11×5=55

Q: [新问题]
A: [模型模仿示例格式]
```

### Self-Consistency

```
1. 用 CoT 生成多个回答（temperature > 0）
2. 取最常见的最终答案

效果：在数学和推理任务上显著提升
成本：生成多个回答
```

## CoT 的训练

### SFT 训练 CoT

```
训练数据格式：
问题 → 思考过程 → 答案

关键：
- 推理步骤要清晰
- 中间结果正确
- 最终答案与过程一致

代表：OpenAI o1, QwQ 等推理模型
```

### 过程监督 (Process Supervision)

```
不仅奖励正确答案，还奖励正确的中间步骤：

步骤 1: 正确 (+1)
步骤 2: 正确 (+1)
步骤 3: 错误 (-1)
步骤 4: ...

vs 结果监督 (Outcome Supervision)：
只看最终答案是否正确

发现：过程监督更有效，尤其是复杂问题
```

### 强化学习训练推理

```
OpenAI o1 的核心方法：

1. 用 RL 训练模型产生长推理链
2. Reward：答案正确 + 过程合理
3. 模型学会：
   - 分解复杂问题
   - 验证中间结果
   - 发现错误时回溯
   - 尝试多种方法
```

## CoT 的局限

| 局限 | 说明 |
|------|------|
| 不保证正确 | 推理过程看起来合理，但可能有逻辑漏洞 |
| 成本增加 | 生成思考过程增加 token 数 |
| 幻觉风险 | 推理步骤中可能编造事实 |
| 不适合简单任务 |  overhead 大，简单任务直接回答更好 |

## Tree of Thoughts (ToT)

```
CoT 的扩展：不再线性推理，而是树形搜索

1. 分解问题为多个思考步骤
2. 每步生成多个候选
3. 评估每个候选的价值
4. 选择最佳路径继续
5. 可回溯、可探索

适用：需要探索的复杂问题（如 24 点游戏、创意写作）
成本：远高于 CoT
```
