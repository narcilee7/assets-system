# LLM-as-Judge

用 LLM 评估 LLM 是现代 AI 评估的重要范式。它可扩展、成本低，但也有固有局限。

## 基本原理

```
人工评估的问题：
- 慢：一个人一天评几百条
- 贵：需要专业标注员
- 不一致：不同人标准不同

LLM 评估的优势：
- 快：秒级评估
- 一致：相同 prompt 相同标准
- 可扩展：任意规模
```

## 评估模式

### 单点评分

```
Prompt: 
"请评估以下回答的质量，从 1-5 打分。
标准：准确性、完整性、清晰度。

问题：{question}
回答：{response}

分数："
```

### 成对比较

```
Prompt:
"比较以下两个回答，判断哪个更好。

问题：{question}
回答 A：{response_a}
回答 B：{response_b}

更好的回答是："
```

- 通常比单点评分更可靠
- 人类也更擅长做比较而非绝对评分

### 参考对比

```
Prompt:
"将以下回答与参考答案对比，评估准确性。

问题：{question}
参考答案：{reference}
待评估回答：{response}

评估："
```

## 评判模型选择

| 模型 | 可靠性 | 偏见 | 成本 |
|------|--------|------|------|
| GPT-4 | 高 | 有 | 高 |
| Claude | 高 | 有 | 高 |
| 开源大模型 | 中 | 有 | 低 |
| 专用 judge 模型 | 中-高 | 较低 | 低 |

## 已知偏见

| 偏见 | 说明 | 缓解 |
|------|------|------|
| 位置偏见 | 偏好第一个/最后一个回答 | 交换位置多次评估 |
| 长度偏见 | 偏好更长的回答 | 明确惩罚冗余，标准化长度 |
| 自我提升 | 偏好自己生成的内容 | 用不同模型评估 |
| 风格偏见 | 偏好特定格式 | 明确格式标准 |
| 难度偏见 | 对简单问题打分偏高 | 按难度分层评估 |

## 评估框架

| 框架 | 特点 |
|------|------|
| MT-Bench | 多轮对话，按类别评分 |
| AlpacaEval | 与 reference 对比，计算 win rate |
| JudgeLM | 专门训练的评判模型 |
| Prometheus | 开源评判模型，有参考评分 |

## 提高可靠性

```
1. 明确标准 (Rubric)
   - 定义清晰的评分维度
   - 提供评分示例

2. 多轮评估
   - 多次采样取平均
   - 交换位置消除位置偏见

3. 分层评估
   - 简单/中等/困难分别评估
   - 不同类别用不同标准

4. 人机结合
   - LLM 初筛
   - 人工审核边界案例
```

## 局限

```
1. 无法评估模型不知道的知识
2. 对创造性任务评估能力有限
3. 可能被 adversarial 样本欺骗
4. 评判标准本身是主观的
```
