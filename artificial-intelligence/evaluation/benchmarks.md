Benchmark 是衡量模型能力的标尺。选择合适的基准、正确解读结果，是模型开发的关键环节。

## 评估陷阱

### 数据污染

```
问题：测试集内容泄露到训练集
结果：分数虚高

检测：
- n-gram 重叠检测
- 嵌入相似度
- 训练时间 vs 基准发布时间

报告：
- 应主动报告污染检测结果
- 说明去重方法
```

### 指标选择

```
错误：只看单一指标
正确：多维度评估

示例：
- 高 MMLU ≠ 好对话模型
- 高 HumanEval ≠ 好助手
- 需要结合多个基准和人工评估
```

### 统计显著性

```
问题：模型 A 72.5% vs 模型 B 72.3%，差 0.2% 算更好吗？

需要：
- 多次采样（temperature > 0）
- 标准差计算
- 假设检验（p-value）

通常：差异 < 1% 且不稳定时，不宜断言优劣
```
