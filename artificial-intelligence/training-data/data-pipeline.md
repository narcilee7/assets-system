# 数据流水线

数据流水线决定了模型能学到什么。高质量的数据是模型能力的上限，而低质量数据则是模型的下限。

## 流水线概览

```
Raw Data
  → 收集 (Crawl / License / Buy)
  → 去重 (Exact / Fuzzy / MinHash)
  → 清洗 (Language / Quality / Toxicity)
  → 过滤 (Length / Format / Domain)
  → 混合 (Curriculum / Ratio / Upsample)
  → 验证 (Contamination / Benchmark leakage)
  → Training Data
```

## 数据来源

| 来源 | 优点 | 缺点 | 占比参考 |
|------|------|------|---------|
| Common Crawl | 规模大、免费 | 噪声大、质量参差 | 60-80% |
| Web Pages | 多样、实时 | 重复、广告、模板 | - |
| Books | 高质量长文本 | 版权限制 | 5-15% |
| Code (GitHub) | 结构化、逻辑 | 许可证问题 | 10-30% |
| Wikipedia | 事实准确 | 规模有限 | 3-5% |
| Academic Papers | 专业、深度 | 可读性差 | 2-5% |
| Dialog / QA | 对话格式 | 收集难 | 1-5% |

## 去重方法

### 精确去重

```
方法：MD5 / SHA256 哈希
适用：完全相同的文档
效果：去除 10-30% 的精确重复
```

### 近重复去重 (Near-Dedup)

```
方法：MinHash + LSH
原理：
1. 将文档切分为 n-gram shingles
2. 计算 MinHash 签名（多个哈希函数的最小值）
3. LSH 分桶：相似文档大概率落入同一桶
4. 桶内比较 Jaccard 相似度

阈值：通常 Jaccard > 0.8-0.9 视为重复
效果：去除 20-40% 的近重复
```

### 文档内去重

```
方法：检测模板化段落、导航栏、版权声明
效果：去除网页中的重复结构
```

## 质量过滤

### 启发式规则

| 规则 | 说明 |
|------|------|
| 长度过滤 | 去除过短（<100 字符）或过长的文档 |
| 语言检测 | fastText 语言分类器，保留目标语言 |
| 标点比例 | 异常高的标点可能表示垃圾数据 |
| 行重复率 | 高重复行表示模板/垃圾 |
| 停用词比例 | 过低表示非自然语言 |

### 质量评分模型

```
方法：训练分类器判断文档质量

正样本：Wikipedia、Books、精选网页
负样本：随机网页、已知低质量数据
特征：困惑度、语言模型分数、格式特征

代表：
- Gopher 质量分类器
- RefinedWeb 过滤
- C4 过滤规则
```

### 困惑度过滤

```
用一个小语言模型计算文档的困惑度：
-  perplexity 低 → 模型"熟悉" → 可能是常见/高质量内容
-  perplexity 高 → 模型"陌生" → 可能是噪声/低质量内容

注意：perplexity 低的也可能是重复/模板内容
```

## 毒性过滤

| 类型 | 方法 | 挑战 |
|------|------|------|
| 成人内容 | 关键词 + 分类器 | 误判（医学、艺术） |
| 仇恨言论 | 分类器 | 上下文依赖 |
| PII | 正则 + NER | 误删有效信息 |
| 偏见 | 统计检测 | 难以量化 |

## 数据混合 (Mixture)

### 领域比例

```
LLaMA-2 的预训练混合（近似）：
- Common Crawl:    67%
- Code:            15%
- Wikipedia:        4.5%
- Books:            4.5%
- ArXiv:            2.5%
- StackExchange:    2%
- Other:            4.5%
```

### 课程学习 (Curriculum Learning)

```
策略：
1. 早期用简单/通用数据
2. 后期用困难/专业数据

实现：
- 动态调整采样比例
- 或按 epoch 逐步改变混合
```

### 上采样 (Upsampling)

```
问题：高质量数据少，低质量数据多
解决：对高质量数据重复采样

例如：
- 原始比例：Books 2%, Web 80%
- 上采样后：Books 10%, Web 72%
```

## 污染检测

```
目标：确保训练数据不包含测试集/benchmark 内容

方法：
1. n-gram 匹配：检测训练集中是否有 benchmark 的 n-gram
2. 嵌入相似度：检测语义相似段落
3. 人工审查：关键 benchmark 的人工检查

注意：
- 完全消除污染几乎不可能
- 需要报告检测方法和结果
```

## 常见误区

| 误区 | 正解 |
|------|------|
| 数据量越大越好 | ❌ 质量比数量重要，低质数据会拉低性能 |
| 去重会损失多样性 | ⚠️ 精确去重不会，但过度 aggressive 会 |
| 过滤标准越严越好 | ❌ 过于严格会移除有价值的数据 |
| 数据混合比例不重要 | ❌ 比例直接影响模型能力分布（代码能力 vs 常识） |
