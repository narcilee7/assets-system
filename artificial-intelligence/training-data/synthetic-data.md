# 合成数据

合成数据是扩展训练数据、提升特定能力的重要手段。随着高质量自然数据接近枯竭，合成数据变得越来越关键。

## 合成方法

### 1. Self-Instruct

```
方法：用现有模型生成指令-输出对

流程：
1. 准备种子指令（少量人工编写）
2. 用模型生成相似指令
3. 用模型生成对应输出
4. 过滤低质量样本

代表：Alpaca、Vicuna 的训练数据
```

**优点**：快速、低成本、可扩展
**缺点**：质量受基模型限制、可能存在错误传播

### 2. 蒸馏 (Distillation)

```
方法：用强模型（GPT-4/Claude）生成高质量数据

流程：
1. 设计 prompt（定义任务、格式、标准）
2. 调用强模型生成
3. 多轮采样 + 过滤
4. 可能加入人工审核

代表：
- ShareGPT（用户分享的 ChatGPT 对话）
- Orca（GPT-4 生成推理数据）
- WizardLM（Evol-Instruct 进化指令）
```

### 3. 推理数据合成

```
方法：生成需要多步推理的问题和解答

技术：
- 从现有数据集抽取问题，重新生成解答
- 用程序验证答案（数学、代码）
- 拒绝采样：多次生成，只保留正确的

代表：
- GSM8K 扩展（数学）
- CodeContests（代码竞赛）
- OpenWebMath（数学网页）
```

### 4. 多智能体合成

```
方法：多个模型角色扮演，生成对话数据

角色：
- 用户：提出问题
- 助手：回答问题
- 批评者：指出回答的问题
- 修订者：改进回答

优点：多样性高、有自我修正
代表：AutoGen、CAMEL 生成的数据
```

### 5. 代码执行合成

```
方法：用代码生成和验证数据

流程：
1. 生成编程问题
2. 用模型生成代码
3. 执行代码验证正确性
4. 保留通过测试的样本

代表：
- CodeAlpaca
- Self-Instruct for Code
- Code-Feedback（执行反馈改进）
```

## 质量控制

### 过滤策略

| 策略 | 方法 | 效果 |
|------|------|------|
| 规则过滤 | 长度、格式、关键词 | 去除明显垃圾 |
| 模型评分 | 用强模型打分 | 保留高质量 |
| 一致性检查 | 多次采样看是否一致 | 去除不确定样本 |
| 验证器 | 程序/公式验证 | 保证正确性 |
| 去重 | 与已有数据去重 | 防止重复 |

### 质量-数量权衡

```
发现：
- 100K 高质量 > 1M 中等质量
- 但 1M 中等质量 + 100K 高质量 > 只有 100K 高质量

策略：
- 核心能力：高质量自然数据
- 扩展覆盖：合成数据补充
```

## 应用场景

| 场景 | 合成方法 | 效果 |
|------|---------|------|
| 指令遵循 | Self-Instruct | 显著提升 |
| 数学推理 | 程序验证合成 | 显著提升 |
| 代码能力 | 执行反馈合成 | 显著提升 |
| 安全对齐 | 对抗性合成 | 提升鲁棒性 |
| 多语言 | 翻译 + 回译 | 扩展语言覆盖 |
| 工具使用 | 模拟 API 调用 | 学习工具调用 |

## 风险与限制

| 风险 | 说明 | 缓解 |
|------|------|------|
| 模型崩溃 | 用合成数据训练 → 能力退化 | 与自然数据混合、多代过滤 |
| 错误传播 | 基模型的错误被放大 | 验证器、人工审核 |
| 同质化 | 所有模型用相似合成数据 | 多样化基模型、创新合成方法 |
| 版权灰色地带 | 蒸馏可能涉及版权问题 | 明确标注、使用开源模型 |

## 常见误区

| 误区 | 正解 |
|------|------|
| 合成数据可以完全替代自然数据 | ❌ 自然数据提供真实分布，合成数据补充 |
| 蒸馏就是复制 GPT-4 | ❌ 是学习方法，不是复制参数 |
| 合成数据没有价值 | ❌ 在推理、代码、安全领域非常有效 |
| 越多合成数据越好 | ❌ 质量控制和多样性同样重要 |
