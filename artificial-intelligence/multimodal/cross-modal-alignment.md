# 跨模态对齐

跨模态对齐是多模态学习的核心：将不同模态的数据映射到统一的语义空间，使得"语义相近"的内容在空间中"距离相近"。

## 对齐目标

```
模态A空间              统一空间              模态B空间
  猫图    ─────────→   [猫]   ←─────────  "猫"
  狗图    ─────────→   [狗]   ←─────────  "狗"
  猫叫声  ─────────→   [猫]   ←─────────  "meow"
```

## 对齐方法

### 1. 对比学习 (Contrastive Learning)

```
正样本对：(图像_i, 文本_i)   → 相似度最大化
负样本对：(图像_i, 文本_j)   → 相似度最小化

Loss = -log[exp(sim/τ) / (exp(sim/τ) + Σ exp(sim_neg/τ))]
```

**关键要素**：
| 要素 | 选择 | 影响 |
|------|------|------|
| 相似度函数 | 余弦相似度、点积 | 归一化 vs 幅度敏感 |
| 温度参数 τ | 0.01-0.1 | 控制分布锐度 |
| 负样本数 |  batch size | 越大越好，但受显存限制 |
| 难负样本挖掘 | 选择相似度高的负样本 | 提升学习效率 |

### 2. 匹配预测 (Matching Prediction)

```
ITM (Image-Text Matching):
  输入：[CLS] 文本 [SEP] 图像特征
  输出：二分类（匹配/不匹配）

需要构造负样本：
  - 随机替换（简单）
  - 相似样本替换（困难）
```

### 3. 掩码预测 (Masked Modeling)

```
MLM + MIM (Masked Language/Image Modeling):
  遮蔽部分文本 token 和图像 patch
  模型需要利用另一模态的信息来重建

强制模型学习跨模态关联
```

### 4. 生成式对齐

```
Captioning: 图像 → 生成描述文本
Image Generation: 文本 → 生成图像

通过生成任务建立细粒度对齐
```

## 对齐质量评估

| 指标 | 方法 | 含义 |
|------|------|------|
| 检索精度 | Image→Text, Text→Image Recall@K | 空间对齐质量 |
| Zero-shot 分类 | 用文本描述做类别 | 语义对齐程度 |
| 线性探测 | 冻结 encoder，训练分类头 | 表示质量 |
| 可视化 | t-SNE/UMAP 降维观察 | 直观理解对齐 |

## 对齐的挑战

| 挑战 | 说明 | 解决方向 |
|------|------|---------|
| 模态鸿沟 | 图像连续，文本离散，结构差异大 | 渐进式对齐、中间表示 |
| 粒度不匹配 | 图像含多个对象，文本是序列 | 局部-全局对齐 |
| 数据噪声 | 图文对不完全匹配（如网页数据） | 噪声鲁棒损失、数据过滤 |
| 负样本质量 | 随机负样本太简单 | 难负样本挖掘 |

## 典型模型对齐策略

| 模型 | 对齐方法 | 特点 |
|------|---------|------|
| CLIP | 对比学习 | 简单高效，双塔 |
| ALBEF | 对比 + ITM + MLM | 多目标联合 |
| BLIP | 对比 + 生成 + 过滤 | 引导式数据过滤 |
| ImageBind | 多模态对比链 | 图像作为枢纽 |

## 常见误区

| 误区 | 正解 |
|------|------|
| 对齐后模态就完全等价了 | ❌ 只是语义空间相近，模态特有信息可能丢失 |
| 对比学习只需要正样本 | ❌ 负样本的质量和数量同样关键 |
| 更大的 batch 总是更好 | ⚠️ 是，但需配合合适的 τ |
| 对齐只发生在训练初期 | ❌ 多模态 SFT 阶段仍在精调对齐 |
