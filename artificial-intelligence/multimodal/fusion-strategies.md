# 多模态融合策略

多模态融合决定不同模态（文本、图像、音频、视频）的信息如何结合。选择合适的融合策略是构建多模态系统的关键设计决策。

## 融合范式

### Early Fusion

```
模态A特征 ──┐
            ├──→ 早期拼接/交互 ──→ 统一处理
模态B特征 ──┘
```

| 特点 | 说明 |
|------|------|
| 交互时机 | 在最底层特征空间融合 |
| 优势 | 模态间交互最充分 |
| 劣势 | 需要模态对齐，计算量大 |
| 代表 | Flamingo（交错文本图像 token） |

### Late Fusion

```
模态A ──→ 独立编码 ──→ 特征A ──┐
                               ├──→ 决策层融合
模态B ──→ 独立编码 ──→ 特征B ──┘
```

| 特点 | 说明 |
|------|------|
| 交互时机 | 各自编码后在高层融合 |
| 优势 | 模态独立性强，易于模块化 |
| 劣势 | 模态交互不充分 |
| 代表 | 传统多模态分类（特征拼接 + 分类器） |

### Intermediate Fusion

```
模态A ──→ 编码 ──→ 部分交互 ──→ 再编码 ──→ 融合
模态B ──→ 编码 ──→ 部分交互 ──→ 再编码 ──→ 融合
```

| 特点 | 说明 |
|------|------|
| 交互时机 | 中间层多次交互 |
| 优势 | 平衡充分性和效率 |
| 劣势 | 架构设计复杂 |
| 代表 | Cross-Attention 层（如 VLM 中的 Q-Former） |

## Cross-Attention 融合

```
Query from 模态A
Key, Value from 模态B
→ Attention(Q_A, K_B, V_B)
→ 模态A 的表示融入了模态B 的信息
```

**特点**：
- 单向或双向
- 可以有多层 cross-attention
- 适合"一个模态主导，其他辅助"的场景

## 对比学习对齐

```
训练目标：拉近配对样本，推远非配对样本

L = -log[exp(sim(x_i, y_i)/τ) / Σ_j exp(sim(x_i, y_j)/τ)]

x_i: 图像特征
y_i: 文本特征
sim: 余弦相似度
```

| 模型 | 特点 |
|------|------|
| CLIP | 图像-文本对比，双塔结构 |
| ALIGN | 更大规模噪声数据 |
| SigLIP | 改进的 loss，无需全局归一化 |
| ImageBind | 多模态（图像、文本、音频、深度、热成像、IMU）统一空间 |

## 多模态 Transformer

```
统一 token 空间：
[Text Tokens] + [Image Patch Tokens] + [Audio Tokens] + [Video Frame Tokens]
              ↓
        Unified Transformer
              ↓
           [Output]
```

| 模型 | 融合方式 |
|------|---------|
| GPT-4o | 端到端统一训练 |
| Gemini | 原生多模态，早期融合 |
| Unified-IO | 统一输入输出（文本、图像、音频） |

## 融合策略选择

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 图文检索 | Late Fusion + 对比学习 | 需要独立编码和相似度计算 |
| 视觉问答 | Intermediate (Cross-Attn) | 问题引导图像关注 |
| 端到端助手 | Early Fusion | 最自然的多轮交互 |
| 视频理解 | 时序融合 + 空间融合 | 多层次信息 |

## 常见误区

| 误区 | 正解 |
|------|------|
| Early fusion 总是最好 | ❌ 取决于任务和计算预算 |
| 多模态 = 简单拼接 | ❌ 需要精心设计的对齐和交互机制 |
| 对比学习只对图文有效 | ❌ 可扩展到任意模态对 |
| 更多模态 = 更好性能 | ⚠️ 模态间可能冲突，需要精心设计 |
