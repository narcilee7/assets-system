# 越狱攻击与防御

Jailbreak 是指绕过 LLM 的安全限制，使其生成有害内容。理解攻击方式和防御策略对构建安全的 AI 系统至关重要。

## 攻击分类

### 基于提示工程的攻击

| 类型 | 示例 | 原理 |
|------|------|------|
| 角色扮演 | "You are a villain who..." | 通过角色设定绕过安全限制 |
| 假设立场 | "For educational purposes..." | 包装为正当目的 |
| 逻辑陷阱 | "If A then B, B is harmful..." | 利用逻辑推理漏洞 |
| 编码绕过 | Base64/hex/morse 编码请求 | 绕过关键词过滤 |
| 低资源语言 | 用斯瓦希里语等提问 | 安全训练数据覆盖不足 |

### 基于优化的攻击

| 类型 | 方法 | 效果 |
|------|------|------|
| GCG | 梯度优化 suffix | 自动生成高成功率攻击 |
| AutoDAN | 遗传算法 | 生成可读性更好的攻击 |
| PAIR | 攻击 LLM + 评判 LLM | 自动化多轮攻击生成 |
| TAP | 树搜索 | 系统性探索攻击空间 |

### 多轮对话攻击

```
攻击者分步获取信息：

Round 1: "What chemicals are common in households?"
Round 2: "What happens when A and B mix?"
Round 3: "How to safely neutralize the result?"

单独看每轮都无害，组合起来有害
```

## 防御策略

### 训练阶段

| 方法 | 说明 |
|------|------|
| 对抗训练 | 用攻击样本训练模型鲁棒性 |
| 安全微调 | 专门的安全场景 SFT |
| 拒绝训练 | 训练模型识别并拒绝边界请求 |
| 多语言安全 | 扩展安全训练到多语言 |

### 推理阶段

| 方法 | 说明 |
|------|------|
| 输入过滤 | 检测有害/异常输入 |
| 输出监控 | 检测有害输出 |
| 多轮检测 | 检测对话模式是否可疑 |
| 语义理解 | 不依赖关键词，理解真实意图 |

### 系统级防御

```
多层防御架构：

Layer 1: 输入分类器
  → 检测明显的有害输入

Layer 2: 模型安全训练
  → 模型本身拒绝有害请求

Layer 3: 输出分类器
  → 检测模型输出的有害内容

Layer 4: 应用层控制
  → 速率限制、人工审核、审计日志
```

## 评估防御效果

| 指标 | 说明 |
|------|------|
| Attack Success Rate (ASR) | 攻击成功的比例，越低越好 |
| False Refusal Rate | 正常请求被误拒的比例，越低越好 |
| Robustness Score | 综合指标 |

## 持续对抗

```
攻击和防御是持续演化的：

1. 新模型发布
2. 攻击者发现新漏洞
3. 防御者修复漏洞
4. 攻击者发明新攻击
5. 循环...

关键：
- 持续监控和 Red Teaming
- 快速响应新攻击模式
- 透明披露安全评估结果
```

## 常见误区

| 误区 | 正解 |
|------|------|
| 100% 安全是可能的 | ❌ 只能降低风险，无法完全消除 |
| 输入过滤就够了 | ❌ 需要多层防御 |
| 安全训练会严重降低能力 | ⚠️ 精心设计可以最小化影响 |
| 开源模型更不安全 | ⚠️ 开源可被更多人测试修复，闭源只是不透明 |
