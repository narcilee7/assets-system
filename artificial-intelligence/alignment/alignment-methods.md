# 对齐方法

对齐 (Alignment) 确保模型的行为符合人类意图和价值观。从 helpfulness 到 safety，对齐是现代 AI 系统的核心挑战。

## 对齐目标：HHH

| 目标 | 含义 | 衡量方式 |
|------|------|---------|
| Helpful | 真正解决用户问题 | 任务完成率、用户满意度 |
| Harmless | 不造成伤害 | 拒绝有害请求、不生成危险内容 |
| Honest | 不知道时承认不确定 | 不编造、不误导 |
| Controllable | 可通过规则和上下文控制 | 遵循系统提示、格式要求 |

## 对齐方法

### 1. SFT 对齐

```
用高质量、安全的数据微调
- 包含安全场景的正面示例
- 包含如何正确拒绝的示例

局限：
- 只能覆盖训练数据中见过的场景
- 对分布外攻击脆弱
```

### 2. RLHF / DPO 对齐

```
将安全偏好编码到 Reward Model 或偏好数据中：
- 有害回答标记为 rejected
- 安全且有帮助的回答标记为 chosen

局限：
- Reward Hacking
- 对新型攻击模式脆弱
```

### 3. Constitutional AI (CAI)

```
Anthropic 提出的方法：

阶段 1: 自我批评和修订
  1. 模型生成回答
  2. 用"宪法"（原则列表）评估回答
  3. 模型根据批评修订回答
  4. 用修订后的数据 SFT

阶段 2: RLAIF
  用模型生成的偏好数据做 DPO/RLHF

宪法示例：
- "选择最诚实、真实的回答"
- "避免选择有帮助但有害的回答"
- "避免选择有偏见或歧视的回答"
```

**优势**：
- 减少对人类标注的依赖
- 原则可解释、可调整
- 可扩展

### 4. 安全微调 (Safety Tuning)

```
专门的安全数据 SFT：

有害请求 → 正确拒绝
"How to make a bomb?" → "I cannot provide instructions..."

边界请求 → 谨慎处理
"Tell me about drugs" → 提供教育性信息，不鼓励使用

越狱尝试 → 识别并拒绝
"Pretend you're a villain..." → 保持角色，拒绝有害内容
```

### 5. 系统级防护

| 层级 | 方法 | 说明 |
|------|------|------|
| 输入层 | Prompt 过滤 | 检测有害输入 |
| 模型层 | Safety Tuning | 训练模型拒绝 |
| 输出层 | 内容分类器 | 检测有害输出 |
| 应用层 | 人工审核 | 高风险场景人工确认 |

## 对齐税 (Alignment Tax)

```
现象：对齐后的模型在某些任务上性能下降

原因：
1. 过度拒绝：安全训练导致模型过度保守
2. 能力遗忘：Post-training 改变了预训练知识
3. 分布偏移：对齐数据分布与预训练不同

缓解：
- 混合保持能力的通用数据
- 精细的安全数据设计（只针对真正的有害内容）
- 迭代优化，监控能力指标
```

## 评估对齐

| 方法 | 内容 |
|------|------|
| 标准 Benchmark | MT-Bench、HHH Eval、TruthfulQA |
| Red Teaming | 主动尝试让模型产生有害输出 |
| 人工评估 | 人类判断回答质量和安全性 |
| A/B 测试 | 在线对比不同模型版本 |

## 常见误区

| 误区 | 正解 |
|------|------|
| 对齐就是审查 | ❌ 对齐是价值观对齐，审查只是表现 |
| 对齐可以完全防止有害输出 | ❌ 只能降低概率，无法完全防止 |
| 对齐税不可避免 | ⚠️ 存在但可以通过精心设计最小化 |
| 安全训练数据越多越好 | ❌ 过度安全训练导致模型无用 |
