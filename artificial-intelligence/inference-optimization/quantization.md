# 量化 (Quantization)

量化将模型权重和/或激活从高精度（FP32/FP16）转为低精度（INT8/INT4），减少显存和计算需求。

## 量化类型

### 权重量化 (W-only)

```
只量化权重，激活保持 FP16

FP16 权重 → INT8/INT4 权重
推理时：反量化为 FP16 → 计算 → 量化回去

最简单，性能影响小
代表：LLM.int8()、GPTQ、AWQ
```

### 权重-激活量化 (W8A8 / W4A16)

```
同时量化权重和激活

需要：
- 校准数据确定缩放因子
- 特殊 kernels 做低精度计算

更高压缩率，但实现复杂
代表：SmoothQuant、LLM-QAT
```

### KV Cache 量化

```
量化 KV Cache 中的 K 和 V

效果：
- 减少 KV Cache 显存占用 2-4×
- 允许更长上下文或更大 batch

挑战：
- 不同 head、layer 的分布差异大
- 需要 per-channel / per-token 量化

代表：KVQuant、KIVI
```

## 量化方法

### GPTQ

```
基于 OBS (Optimal Brain Surgeon) 的逐层量化：

1. 逐层处理
2. 用 Hessian 矩阵信息补偿量化误差
3. 将误差传播到未量化的权重

特点：
- 4-bit 量化，精度损失小
- 适合静态部署
- 量化过程较慢（离线）
```

### AWQ (Activation-aware Weight Quantization)

```
核心洞察：不是所有权重同等重要
- 保护"激活值大"对应的权重（对输出影响大）
- 通过逐通道缩放实现

特点：
- 4-bit 量化
- 比 GPTQ 更快（无需重构）
- 精度通常优于 GPTQ
```

### GGUF / llama.cpp

```
面向 CPU/边缘设备的量化：
- Q4_0, Q4_1, Q5_0, Q5_1, Q8_0 等多种格式
- 支持混合精度（不同层不同精度）
- 针对 ARM/x86 优化

适用：本地运行、移动设备、低资源环境
```

## 精度与压缩对比

| 方法 | 权重位宽 | 激活位宽 | 显存压缩 | 精度损失 | 速度提升 |
|------|---------|---------|---------|---------|---------|
| FP16 | 16 | 16 | 1× | 基准 | 基准 |
| INT8 (W8A16) | 8 | 16 | 2× | 极小 | 1.5-2× |
| INT8 (W8A8) | 8 | 8 | 2× | 小 | 2-3× |
| INT4 (GPTQ/AWQ) | 4 | 16 | 4× | 中 | 2-3×* |
| INT4 (GGUF) | 4 | 16/32 | 4× | 中-大 | 1-2× (CPU) |

*需要专用 kernel 才能实现加速

## 量化感知训练 (QAT)

```
在训练中加入量化模拟：
1. 前向时模拟量化（伪量化）
2. 反向传播用 STE (Straight-Through Estimator) 近似梯度
3. 模型学会"适应"量化

效果：比训练后量化更好，但训练成本高
适用：从头训练或大规模微调
```

## 常见误区

| 误区 | 正解 |
|------|------|
| 量化总是加速 | ❌ 只有配合专用 kernel 才加速，否则可能更慢 |
| 4-bit 和 8-bit 精度差异不大 | ⚠️ 对复杂任务（推理、代码）差异明显 |
| 所有层应该统一量化 | ❌ 敏感层（如 embedding）可用更高精度 |
| 量化后不能微调 | ❌ QLoRA 可以在 4-bit 模型上高效微调 |
