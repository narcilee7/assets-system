# Speculative Decoding

Speculative Decoding 用小型草稿模型加速大型目标模型的生成，在保持输出分布不变的前提下实现 2-3× 加速。

## 核心思想

```
问题：大模型生成每个 token 都很慢
观察：很多 token 是"容易"的（确定性高）

想法：用小模型快速生成草稿，大模型验证

草稿模型 (小，快)          目标模型 (大，慢)
  → 生成 K 个 draft tokens  → 一次并行验证
  ← 接受 N 个              → 修正第 N+1 个
```

## 算法流程

```
1. 草稿模型自回归生成 K 个 tokens：d_1, d_2, ..., d_K

2. 目标模型并行验证：
   输入：[prefix] + d_1 + d_2 + ... + d_K
   输出：p_1, p_2, ..., p_K+1
   
   p_i = P(target | prefix + d_1 + ... + d_{i-1})

3. 逐个接受/拒绝：
   对于 i = 1 to K:
     采样 u ~ Uniform(0,1)
     如果 u < min(1, p_i(d_i) / q_i(d_i)):
        接受 d_i
     否则：
        从修正分布采样新 token，停止验证

4. 如果没有全部接受，从修正分布采样剩余部分
```

## 关键性质

```
数学保证：
- 输出分布与目标模型完全一致
- 是无损加速！

接受概率 ≈ E[min(1, p/q)]
≈ 草稿模型和目标模型的一致性
```

## 草稿模型选择

| 策略 | 草稿模型 | 效果 |
|------|---------|------|
| 独立小模型 | 单独训练的 1-2B 模型 | 好，但需维护两个模型 |
| 自身小版本 | 同一系列的更小版本 | 好，分布相近 |
| 共享模型 | 用目标模型的浅层 | 省显存，效果一般 |
| 递归推测 | 多级草稿（大→中→小） | 进一步加速 |

## 推测采样变体

### Medusa

```
不用草稿模型，而是训练多个解码头：
- 主头：预测下一个 token
- 附加头：预测下 2nd, 3rd, ... K-th token

优势：无需第二个模型
劣势：需要训练，占用额外参数
```

### Lookahead Decoding

```
不用草稿模型，用输入本身的 n-gram 做草稿：
- 在已生成的文本中找重复模式
- 用 Jacobi 迭代并行验证

优势：无需额外模型或训练
劣势：对非重复内容加速有限
```

### Eagle

```
在特征层做推测：
- 用轻量网络从当前隐藏状态预测下一层特征
- 比 token 级推测更细粒度
```

## 效率分析

```
理想情况：
- 草稿模型速度 = 5× 目标模型
- 接受率 = 80%
- K = 5

效果：
- 每步平均生成 1 + 0.8×5 = 5 tokens（等效）
- 实际速度提升 ≈ 2-3×（验证也有开销）
```

## 适用场景

| 场景 | 效果 | 原因 |
|------|------|------|
| 代码生成 | 很好 | 代码有很多确定性模式 |
| 重复性文本 | 很好 | 模板化内容 |
| 创意写作 | 一般 | 高熵，接受率低 |
| 短序列 | 有限 | 启动开销占比大 |

## 常见误区

| 误区 | 正解 |
|------|------|
| Speculative Decoding 改变输出 | ❌ 数学保证分布一致 |
| 草稿模型越大越好 | ❌ 需要速度和质量平衡 |
| 对所有任务都有效 | ❌ 高熵任务加速有限 |
| Medusa 总是比独立草稿好 | ⚠️ 各有优劣，Medusa 省显存但需训练 |
