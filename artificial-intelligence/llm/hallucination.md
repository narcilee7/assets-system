# LLM 幻觉问题

幻觉 (Hallucination) 是 LLM 生成看似合理但实际错误内容的现象。它是制约 LLM 可靠性的核心问题。

## 幻觉类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 事实性幻觉 | 编造不存在的事实 | "爱因斯坦获得了诺贝尔文学奖" |
| 忠实性幻觉 | 输出与输入/上下文矛盾 | 摘要中包含原文没有的信息 |
| 引用幻觉 | 编造不存在的来源 | "根据 2023 年《自然》杂志..." |
| 逻辑幻觉 | 推理过程有漏洞 | "所有鸟都会飞，企鹅是鸟，所以企鹅会飞" |

## 产生原因

### 模型层面

```
1. 训练目标限制
   - 只优化 next token 的似然
   - 不区分"知道"和"猜测"

2. 知识边界模糊
   - 模型没有显式的"我不知道"能力
   - 低概率 token 仍可能被采样

3. 记忆 vs 推理混淆
   - 模型依赖模式匹配而非真正理解
   - 相似问题可能触发错误知识

4. 对齐阶段的副作用
   - 过度追求"有帮助"可能导致编造
   - Safety tuning 可能降低事实准确性
```

### 数据层面

| 问题 | 影响 |
|------|------|
| 训练数据过时 | 对近期事件一无所知或错误 |
| 数据中的错误 | 错误信息被模型学习 |
| 数据重复 | 流行错误被过度表示 |
| 缺乏否定训练 | 模型不擅长表达不确定 |

## 检测方法

| 方法 | 原理 | 局限 |
|------|------|------|
| Self-Consistency | 多次采样，看答案是否一致 | 成本高，系统错误检测不到 |
| Chain-of-Verification | 让模型验证自己的输出 | 验证本身可能出错 |
| 外部检索 | 与知识库对比 | 只能检测有来源的事实 |
| Uncertainty Estimation | 看 token 概率分布 | 概率低 ≠ 错误，概率高 ≠ 正确 |
| 专用检测器 | 训练分类器判断 | 需要标注数据，泛化有限 |

## 缓解策略

### 训练阶段

| 方法 | 原理 |
|------|------|
| 高质量数据过滤 | 去除低质量、矛盾的训练样本 |
| 知识边界训练 | 训练模型识别"我不知道" |
| 检索增强预训练 | 训练时结合检索结果 |
| 事实性 SFT | 用高质量事实数据微调 |

### 推理阶段

| 方法 | 原理 | 成本 |
|------|------|------|
| RAG | 检索相关文档再生成 | 中（检索延迟） |
| Tool Use | 调用搜索/API 验证 | 高（API 调用） |
| Self-Consistency | 多次采样投票 | 高（多次推理） |
| Constrained Decoding | 限制输出格式/来源 | 低 |
| 降低 Temperature | 减少随机性 | 低 |

### 后处理

| 方法 | 原理 |
|------|------|
| 引用验证 | 提取引用，验证是否存在 |
| 事实检查 API | 调用外部事实检查服务 |
| 人机审核 | 关键内容人工确认 |

## 评估

| 基准 | 内容 |
|------|------|
| TruthfulQA | 对抗性设计的问题，诱导模型说谎 |
| HaluEval | 多领域幻觉检测 |
| FACTOR | 评估模型对自己知识的认知 |
| FActScore | 细粒度事实精度评估 |

## 常见误区

| 误区 | 正解 |
|------|------|
| 幻觉可以完全消除 | ❌ 只能缓解，无法完全消除 |
| 大模型幻觉更少 | ⚠️ 不一定，大模型可能更"自信"地编造 |
| RAG 能解决所有幻觉 | ❌ RAG 只能缓解事实性幻觉，逻辑/忠实性幻觉仍存在 |
| 低 temperature 就没有幻觉 | ❌ 减少随机性，但模型仍可能" confidently wrong" |
